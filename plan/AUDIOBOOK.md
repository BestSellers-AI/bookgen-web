# Audiobook Addon (ADDON_AUDIOBOOK)

## Overview

The audiobook addon generates a full narrated audiobook from a book's content using **Edge TTS** (Microsoft Azure via `node-edge-tts`). It costs **60 credits** and works for both original and translated books.

- Engine: Edge TTS (Microsoft Azure neural voices)
- Audio format: MP3, 24kHz, 48kbit/s, mono
- Output: Per-segment audio files + concatenated full audiobook
- Storage: AWS S3
- Free regeneration supported (cost 0 with `params.regenerate = true`)

| `es-MX` | Dalia (es-MX-DaliaNeural) | Jorge (es-MX-JorgeNeural) |

| `de` | Katja (de-DE-KatjaNeural) | Conrad (de-DE-ConradNeural) |
| `it` | Elsa (it-IT-ElsaNeural) | Diego (it-IT-DiegoNeural) |

Fallback: `en` voices if language code is not found. Resolution tries exact match, then base language (e.g., `pt-BR` -> `pt`).

### Audio Format

- Output format: `audio-24khz-48kbitrate-mono-mp3`
- Timeout: 120 seconds per chunk

### Chunking

Edge TTS has a ~3000 character limit per request. The service splits at ~2500 characters (`MAX_CHUNK_CHARS`), breaking at paragraph boundaries first, then sentence boundaries if a paragraph exceeds the limit.

### Duration Extraction

1. Primary: Parse subtitle JSON file generated by Edge TTS, take `end` time of last entry
2. Fallback: Estimate via `Math.ceil(buffer.length / 6000)` when subtitle data unavailable

## Content Preparation

### HTML Stripping

Removes HTML tags and entities before TTS narration:
- `<br>` -> newline, `</p>` -> double newline
- Strips all remaining HTML tags
- Decodes entities: `&nbsp;`, `&amp;`, `&lt;`, `&gt;`, `&quot;`, `&#39;`

### Markdown Stripping

Removes all Markdown formatting to produce clean plain text:
- Headings (`## Title` -> `Title`)
- Bold (`**text**` / `__text__` -> `text`)
- Italic (`*text*` / `_text_` -> `text`)
- Strikethrough (`~~text~~` -> `text`)
- Code blocks (triple backticks -> removed entirely)
- Inline code (backticks -> `text`)
- List markers (unordered `- ` and ordered `1. ` -> removed)
- Blockquotes (`> ` -> removed)
- Links (`[text](url)` -> `text`)
- Images (`![alt](url)` -> `alt`)
- Horizontal rules (`---` -> removed)

### Narration Structure

Each segment's title is narrated before the content with a period pause:
```
{cleanTitle}.\n\n{cleanContent}
```

Empty segments are skipped.

## Segments Included

Segments are built in the following order:

1. **Introduction** (if exists)
2. **Chapters** (sequential, ordered by `sequence`)
3. **Conclusion** (if exists)
4. **Final Considerations** (if exists)
5. **Glossary** (if exists)
6. **Resources and References** (if exists, original book only)
7. **Appendix** (if exists)
8. **Closure** (if exists)

### Section Labels

Section labels are localized and hardcoded in the processor:

| Section | English | Portuguese (BR) | Spanish |
|---------|---------|-----------------|---------|
| introduction | Introduction | Introducao | Introduccion |
| conclusion | Conclusion | Conclusao | Conclusion |
| finalConsiderations | Final Considerations | Consideracoes Finais | Consideraciones Finales |
| glossary | Glossary | Glossario | Glosario |
| resourcesReferences | Resources and References | Recursos e Referencias | Recursos y Referencias |
| appendix | Appendix | Apendice | Apendice |
| closure | Closure | Encerramento | Cierre |

### Chapter Content Fallback

For original books, chapter content is resolved in order:
1. `editedContent` (user-edited version)
2. `content` (generated content)
3. `topics` (fallback: joins topic title + content pairs)

For translated books, `translatedContent` is used directly, with `targetLanguage` determining the voice.

## Generation Flow

### Request

1. User triggers via `POST /api/books/{bookId}/addons` with:
   ```json
   {
     "kind": "ADDON_AUDIOBOOK",
     "params": {
       "voiceGender": "female" | "male",
       "translationId": "optional-translation-id"
     }
   }
   ```
2. Backend (`AddonService.request()`): validates book, debits credits, dispatches BullMQ job

### Processing (`processAddonAudiobook` in `generation.processor.ts`)

1. **Resolve language + segments**: Fetches book or translation data, builds ordered segment list
2. **Resolve voice**: `ttsService.getVoiceForLanguage(language, voiceGender)` -> `{ voiceId, voiceName }`
3. **Generate per-segment audio** (5% -> 85% progress):
   - Strip HTML tags and Markdown from title + content
   - Synthesize via `ttsService.synthesize(plainText, voiceId)`
   - Upload to S3
   - Emit SSE progress event (`book.addon.progress`) per segment
   - Heartbeat (`job.updateProgress()`) between segments
4. **Concatenate full audiobook** (85% -> 95%):
   - `Buffer.concat(allBuffers)` -> upload to S3 as full audiobook
5. **Complete** (95% -> 100%):
   - `hooksService.processAddonResult()` with result data containing all segment URLs + full audio URL

### S3 Paths

| Type | Path Pattern |
|------|-------------|
| Original book segment | `audiobooks/{bookId}/{sectionType|chapter-N}-{timestamp}.mp3` |
| Translation segment | `audiobooks/{bookId}/tr-{translationId}/{sectionType|chapter-N}-{timestamp}.mp3` |
| Original full audiobook | `audiobooks/{bookId}/full-{timestamp}.mp3` |
| Translation full audiobook | `audiobooks/{bookId}/tr-{translationId}/full-{timestamp}.mp3` |

## Database

### Audiobook Model

```
Audiobook (audiobooks)
├── id, bookId, translationId?
├── voiceId, voiceName
├── status (PENDING/PROCESSING/COMPLETED/ERROR)
├── totalDuration (seconds)
├── fullAudioUrl, fullAudioSize (bytes)
├── createdAt, updatedAt
└── chapters -> AudiobookChapter[]
```

### AudiobookChapter Model

```
AudiobookChapter (audiobook_chapters)
├── id, audiobookId
├── chapterId? (nullable — null for sections like intro/conclusion)
├── sectionType? (introduction, conclusion, finalConsiderations, glossary, resourcesReferences, appendix, closure)
├── sequence, title
├── audioUrl, durationSecs
├── status (PENDING/GENERATED)
└── createdAt
```

## Frontend

### AudiobookViewer (`apps/web/src/components/book/audiobook-viewer.tsx`)

Bottom sheet (`Sheet` from shadcn/ui) with two tabs:

**Chapters Tab:**
- List of all segments (sections + chapters)
- Each row: play/pause button, title, section type or chapter number, duration, download button
- Active chapter highlighted with emerald accent + "Now Playing" badge with pulse animation
- Auto-advance to next chapter when current finishes

**Full Audiobook Tab:**
- Card with headphones icon, total duration, voice name
- Play/Pause + Download buttons
- Progress bar when playing

**Player Bar** (shown when audio is playing):
- Play/pause + prev/next chapter navigation
- Now playing label (truncated)
- Seek bar (clickable)
- Current time / total duration display

**Header:**
- Voice name + total duration info
- Regenerate button (calls `onRegenerate` callback)

**i18n:**
- `audiobook` namespace for player labels (title, chapters, fullAudiobook, nowPlaying, play, pause, download, etc.)
- `book` namespace for section type labels (introduction, conclusion, etc.)

### Download

- Per-segment download via `<a href={audioUrl} download={filename}>` links
- Full audiobook download similarly via direct link

---

## Free Regeneration

- When `params.regenerate = true`, the addon cost is 0 (free)
- Regenerate button is inside the audiobook viewer header
- Clicking it opens the voice selection dialog, allowing the user to pick a different voice gender
- The new request is submitted with the `regenerate` flag
- `getExistingAddon()` prioritizes PROCESSING addons over COMPLETED ones, so the UI shows the in-progress state during regeneration
